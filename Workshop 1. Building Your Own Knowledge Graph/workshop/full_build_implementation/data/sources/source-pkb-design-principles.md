# PKB Design Principles

A personal knowledge base works best when each note has a stable purpose, a predictable structure, and visible source provenance. Notes should be small enough to link, compare, and reuse. The system becomes more useful when metadata makes the collection searchable and when explicit relationships make it navigable as a graph rather than a pile of files.

In beginner-friendly systems, plain text files remain one of the strongest choices because they are portable, inspectable, and easy to version. A simple frontmatter contract gives enough structure for filtering and automation without turning note-taking into a database administration task.
